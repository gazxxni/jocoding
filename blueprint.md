# AI 동물상 테스트 사이트 개발 계획

## 1. 프로젝트 개요

본 프로젝트는 Google의 Teachable Machine으로 생성된 인공지능 모델을 사용하여 사용자의 얼굴이 강아지상에 가까운지, 고양이상에 가까운지를 판별해주는 '동물상 테스트' 웹 애플리케이션입니다. shadcn/ui의 디자인 철학에 영감을 받아, 미니멀하고 모던한 UI를 통해 사용자 경험을 극대화하며, Google AdSense를 통한 광고 수익화를 목표로 합니다.

## 2. 주요 기능 (기존)

-   **AI 모델 연동:** Teachable Machine으로 학습된 강아지/고양이 이미지 분류 모델을 사용합니다.
-   **이미지 파일 업로드:** 사용자가 로컬 장치에서 이미지 파일을 선택하여 업로드할 수 있습니다.
-   **모던 UI/UX:** shadcn/ui 스타일에서 영감을 받은 깔끔하고 세련된 디자인을 적용합니다.
-   **결과 시각화:** 분석 결과를 텍스트 기반의 깔끔한 형태로 제공하여 가독성을 높입니다.
-   **테마 변경:** 라이트 모드와 다크 모드를 모두 지원하며, 각 모드에 최적화된 색상 팔레트를 사용합니다.
-   **Google AdSense 통합:** 페이지에 Google AdSense 광고를 추가하여 수익화를 지원합니다.

## 3. AdSense 승인을 위한 콘텐츠 보강 계획 (현재 진행 계획)

Google AdSense 승인 기준을 충족하고 사용자에게 더 나은 가치를 제공하기 위해, 단순한 도구를 넘어 하나의 완성된 마이크로 사이트로 웹사이트를 확장합니다.

**1. 랜딩 페이지 구조 개편:**
    -   **목표:** 사용자가 사이트에 처음 방문했을 때 명확한 정보와 즐길 거리를 제공하여 사이트의 신뢰도를 높이고 이탈률을 낮춥니다.
    -   **실행 방안:**
        -   **Hero 섹션:** 시선을 사로잡는 헤드라인과 서비스에 대한 흥미로운 설명, 그리고 사용을 유도하는 명확한 CTA(Call-to-Action) 버튼을 배치합니다.
        -   **'How it Works' 섹션:** '이미지 업로드' → 'AI 분석' → '결과 확인'의 3단계 과정을 아이콘과 함께 시각적으로 안내하여 사용법을 쉽게 인지시킵니다.
        -   **'About' 섹션:** 서비스의 목적(예: '재미로 즐기는 AI 기술')과 사용된 기술(Teachable Machine)에 대해 간략히 소개해 전문성과 신뢰감을 더합니다.
        -   **Footer:** 저작권 정보와 함께, 신규로 추가될 '개인정보처리방침' 및 '서비스 약관' 페이지로 이동할 수 있는 링크를 포함합니다.

**2. 신규 콘텐츠 페이지 추가:**
    -   **목표:** 웹사이트의 완전성을 높이고 AdSense 정책에서 요구하는 필수 법적 고지 페이지를 갖춥니다.
    -   **실행 방안:**
        -   **`privacy.html` (개인정보처리방침):** 사용자 데이터 처리 방침을 명시하는 페이지를 생성합니다.
        -   **`terms.html` (서비스 이용약관):** 서비스 사용에 대한 규칙과 조건을 명시하는 페이지를 생성합니다.

**3. 코드 및 디자인 개선:**
    -   **목표:** 최신 웹 표준을 적용하여 코드의 유지보수성을 높이고, 사용자 경험을 한 단계 끌어올립니다.
    -   **실행 방안:**
        -   **웹 컴포넌트 도입:** 기존의 동물상 테스트 기능을 하나의 독립된 `<animal-face-tester>` 웹 컴포넌트로 캡슐화하여 코드의 재사용성과 가독성을 높입니다.
        -   **디자인 폴리싱:** 부드러운 스크롤 애니메이션, 인터랙티브한 요소, 개선된 레이아웃을 적용하여 사이트에 생동감을 부여합니다.
        -   **아이콘 활용:** Font Awesome 등의 라이브러리를 활용해 'How it works' 섹션 등의 정보 전달력을 강화합니다.

**4. SEO 및 접근성 개선:**
    -   **목표:** 검색 엔진 최적화(SEO) 및 웹 접근성(A11Y)을 강화하여 더 많은 사용자가 사이트를 발견하고 쉽게 이용할 수 있도록 합니다.
    -   **실행 방안:**
        -   **FAQ 섹션 추가:** '자주 묻는 질문' 섹션을 추가하여 사용자에게 유용한 정보를 제공하고, 페이지의 콘텐츠를 보강합니다. Schema.org의 `FAQPage` 마크업을 사용하여 SEO 효과를 극대화합니다.
        -   **ARIA 속성 추가:** 스크린 리더 사용자를 위해 네비게이션 요소에 `role="navigation"`을 추가하고, 장식용 아이콘에 `aria-hidden="true"`를 적용하여 웹 접근성을 향상시킵니다. (적용 완료)
        -   **시맨틱 HTML 개선:** 기존 코드 구조를 검토하고, 모든 요소가 시맨틱하게 사용되었는지 확인하여 의미론적 마크업을 강화합니다. (적용 완료)
        -   **동적 콘텐츠 SEO 분석:** `main.js`의 `AnimalFaceTester` 웹 컴포넌트와 같이 클라이언트 측에서 동적으로 생성되는 콘텐츠가 SEO에 미치는 영향을 분석했습니다. 현재 아키텍처에서 사용자 상호작용 없이 동적 결과를 직접 색인화하는 것은 어렵다고 판단하며, 정적 콘텐츠를 최대한 풍부하고 SEO 친화적으로 유지하는 데 집중했습니다.

**5. AdSense 정책 준수 강화 (기존 계획 유지):**
    -   분석 결과가 나온 후에만 광고를 노출하는 현재 로직을 유지하여, 사용자에게 가치 있는 콘텐츠가 제공되는 시점에 광고가 노출되도록 합니다.
